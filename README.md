# Mail Print Agent

Демон для автоматической печати вложений из электронной почты. Периодически опрашивает почтовый ящик по IMAP, скачивает вложения из новых писем, при необходимости конвертирует их в PDF и отправляет на сетевой принтер. После успешной печати вложение сохраняется локально, а письмо удаляется из ящика.

## Требования

- Docker и Docker Compose (Compose v2, команда `docker compose`)
- Сетевой принтер с поддержкой IPP (AirPrint / IPP Everywhere) или RAW-сокета (порт 9100)
- IMAP-аккаунт с поддержкой IMAP (Gmail, Yandex, любой корпоративный сервер)

## Быстрый старт

### 1. Склонировать репозиторий

```bash
git clone <repo-url> mail-print-agent
cd mail-print-agent
```

### 2. Создать файл конфигурации

```bash
cp .env.example .env
```

Открыть `.env` и заполнить обязательные параметры:

```dotenv
IMAP_HOST=imap.gmail.com
IMAP_USER=your@gmail.com
IMAP_PASSWORD=xxxx-xxxx-xxxx-xxxx   # пароль приложения Gmail

PRINTER_NAME=office
PRINTER_URI=ipp://192.168.1.50/ipp/print

ALLOWED_SENDERS=your@gmail.com,trusted@example.com
```

### 3. Создать белый список отправителей (опционально)

```bash
cp allowed_senders.txt.example allowed_senders.txt
# отредактировать allowed_senders.txt
```

Раскомментировать соответствующий volume в `compose.yaml`.

### 4. Запустить

```bash
docker compose up -d --build
docker compose logs -f
```

Демон начнёт опрашивать ящик с интервалом `POLL_INTERVAL` секунд (по умолчанию 60).

## Настройка

Все параметры задаются в файле `.env` (копируется из `.env.example`).

### IMAP

| Переменная | По умолчанию | Описание |
|---|---|---|
| `IMAP_HOST` | — | Хост IMAP-сервера |
| `IMAP_PORT` | `993` | Порт IMAP |
| `IMAP_SSL` | `true` | Использовать SSL/TLS |
| `IMAP_USER` | — | Логин (обычно email) |
| `IMAP_PASSWORD` | — | Пароль или пароль приложения |
| `IMAP_FOLDER` | `INBOX` | Папка для мониторинга |
| `POLL_INTERVAL` | `60` | Интервал опроса (секунды) |

> **Gmail:** включить IMAP в настройках аккаунта, создать «Пароль приложения» в разделе Безопасность (двухэтапная аутентификация обязательна).

### Безопасность вложений

| Переменная | По умолчанию | Описание |
|---|---|---|
| `MAX_ATTACHMENT_MB` | `20` | Максимальный размер вложения (МБ) |
| `ALLOWED_EXTENSIONS` | `pdf,png,jpg,jpeg,docx,doc,xlsx,xls,odt,ods,txt` | Допустимые расширения |

### Обработка вложений

| Переменная | По умолчанию | Описание |
|---|---|---|
| `PRINT_ATTACHMENTS` | `FIRST` | `FIRST` — только первое вложение; `ALL` — все |
| `PRINTED_DIR` | `/data/printed` | Папка для сохранения распечатанных файлов (внутри контейнера) |
| `DELETE_FROM_MAILBOX` | `true` | Удалять письмо после успешной печати |

### Принтер

| Переменная | По умолчанию | Описание |
|---|---|---|
| `PRINTER_NAME` | — | Имя принтера в CUPS |
| `PRINTER_URI` | — | URI принтера (`ipp://...` или `socket://...`) |
| `PRINT_DEFAULT_ORIENTATION` | `P` | Ориентация по умолчанию: `P` (портрет) / `L` (ландшафт) |
| `PRINT_DEFAULT_SIDES` | `2` | Сторонность: `1` (одностороння) / `2` (двусторонняя) |
| `PRINT_DEFAULT_EDGE` | `l` | Край переплёта при двусторонней печати: `l` (длинный) / `s` (короткий) |

**Примеры URI принтера:**

```
ipp://192.168.1.50/ipp/print        # IPP / AirPrint
ipp://192.168.1.50:631/ipp/print    # IPP с явным портом
socket://192.168.1.50:9100          # RAW-сокет (jetdirect)
```

## Код печати в теле письма

Первые **3 непробельных символа** текстовой части письма могут задавать параметры печати для конкретного письма. Это переопределяет дефолты из `.env`.

| Позиция | Значения | Смысл |
|---------|----------|-------|
| 1 | `P` / `L` | Ориентация: Portrait / Landscape |
| 2 | `1` / `2` | Сторонность: одностороння / двусторонняя |
| 3 | `s` / `l` | Край переплёта: short / long (актуально только при `2`) |

**Примеры:**

- `P1s` — портрет, односторонняя печать
- `L2l` — ландшафт, двусторонняя, длинный край
- `P2s` — портрет, двусторонняя, короткий край (буклет)

Регистр символов не важен. Если код невалидный или отсутствует — используются дефолты из `.env` целиком (частичного применения нет).

## Белый список отправителей

Принимаются письма **только от адресов из белого списка**. Это обязательная защита от нежелательной печати.

Белый список формируется из двух источников (объединяются):

1. **Переменная окружения** `ALLOWED_SENDERS` — адреса через запятую:
   ```dotenv
   ALLOWED_SENDERS=alice@example.com,bob@example.com
   ```

2. **Файл** `allowed_senders.txt` (опционально). Нужно раскомментировать volume в `compose.yaml`:
   ```yaml
   volumes:
     - ./allowed_senders.txt:/app/allowed_senders.txt:ro
   ```
   И указать в `.env`:
   ```dotenv
   ALLOWED_SENDERS_FILE=/app/allowed_senders.txt
   ```
   Формат файла: один адрес на строку, строки начинающиеся с `#` — комментарии.

Адреса сравниваются без учёта регистра.

## Просмотр логов и статуса

```bash
# Следить за логами в реальном времени
docker compose logs -f

# Последние 100 строк
docker compose logs --tail=100

# Просмотр базы обработанных писем
sqlite3 data/state.db "SELECT message_id, sender, subject, status, processed_at FROM processed ORDER BY processed_at DESC LIMIT 20;"

# Просмотр сохранённых файлов
ls -lh printed/
```

Значения поля `status` в БД:

| Статус | Описание |
|--------|----------|
| `printed` | Все вложения успешно распечатаны |
| `partial` | Часть вложений распечатана, часть — с ошибкой |
| `rejected` | Письмо отклонено (нет допустимых вложений) |
| `error` | Все вложения завершились с ошибкой |

## Устранение неполадок

**Принтер не найден / ошибка lp:**
- Проверить доступность принтера: `ping 192.168.1.50` из контейнера
- Проверить URI в `.env` (`PRINTER_URI`)
- Посмотреть логи CUPS внутри контейнера: `docker compose exec mail-print-agent lpstat -p`

**Письма не забираются:**
- Убедиться, что в `.env` верный пароль (для Gmail — пароль приложения, не основной)
- Убедиться, что IMAP включён в настройках почтового сервиса
- Проверить `ALLOWED_SENDERS` — отправитель должен быть в списке

**LibreOffice не конвертирует файл:**
- Проверить логи: `docker compose logs | grep -i libreoffice`
- Убедиться, что расширение файла входит в `ALLOWED_EXTENSIONS`

**Письмо печатается повторно:**
- Этого не должно происходить — дедупликация ведётся по `Message-ID` в `data/state.db`
- Проверить: `sqlite3 data/state.db "SELECT * FROM processed WHERE message_id='...'"`
